# Migrate Envoy LoadBalancer VIPs from Cilium L2 to BGP

## Context

After the Kubernetes v1.36.0 / kubelet rolling upgrade via Tuppr, `network/envoy-internal` (`10.0.80.200`) returned `connection refused` even though Envoy pods, Services, EndpointSlices, HTTPRoutes, and Gateway status looked healthy.

Root cause: Cilium L2 lease `cilium-l2announce-network-envoy-internal` was held by `k8s-node-2`, but `envoy-internal` uses `externalTrafficPolicy: Local` and had no local ready Envoy endpoint on that node. Deleting the lease forced re-election to a node with a local endpoint and restored service.

Cilium docs warn L2 announcements are incompatible with `externalTrafficPolicy: Local`; Cilium BGP advertisements respect local endpoint presence for `Local` Services.

## Goal

Stage a low-risk migration inspired by onedr0p's Cilium/UniFi BGP setup, but scoped initially to only the Envoy Gateway VIPs:

- `network/envoy-internal` → `10.0.80.200/32`
- `network/envoy-external` → `10.0.80.201/32`

Keep Cilium L2 enabled during initial testing; do not advertise all LoadBalancer services or Kubernetes API yet.

## Rollout outline

1. Ship/verify alert for Cilium L2 holder without local Envoy endpoint.
2. Prepare UniFi/UCG BGP config using ASN `64513` on UCG and `64514` for Cilium nodes, with inbound prefix-list limited to the two Envoy VIPs.
3. Upload UniFi BGP config via Settings → Policy Engine → BGP.
4. Add Cilium v2 BGP resources for the UCG peer and Envoy-only LoadBalancerIP advertisement.
5. Verify BGP peers/routes and service reachability.
6. Observe for 24–48h.
7. Later decide whether to disable L2 for Envoy or move to a dedicated routed VIP range.

## Rollback principle

At almost every stage, disable/remove the UniFi BGP config and leave existing Cilium L2 announcements in place. If L2 lands on a bad holder again, delete only the affected `cilium-l2announce-network-envoy-*` lease after confirmation.

## Progress notes

### 2026-05-09 — Routed Envoy prefix trial

Moved Envoy Gateway VIPs toward an onedr0p-style routed LoadBalancer prefix:

- Added Cilium LB IPAM block `10.0.88.0/24`
- Moved `network/envoy-internal` from `10.0.80.200` to `10.0.88.200`
- Moved `network/envoy-external` from `10.0.80.201` to `10.0.88.201`
- Envoy remains excluded from Cilium L2 announcements and should be reachable via UCG BGP `/32` routes only

Keep BGP scope Envoy-only until UniFi inbound route filtering is stronger and the Cilium BGP readiness race is fixed or accepted operationally.

### 2026-05-09 — Monitoring and persisted UCG route filtering

- Re-uploaded UCG BGP configuration through UniFi with persistent exact `/32` inbound prefix-list filtering for `10.0.88.200/32` and `10.0.88.201/32`.

Observation phase remains active for 24–48h. Do not expand BGP beyond Envoy during this window.
